# Issue #13: 配送結果画面の実装

### 背景 / 目的
配送終了後にユーザーに結果を分かりやすく表示し、達成感を与えるモーダル画面を実装する。

- **依存**: #12
- **ラベル**: `frontend`, `feature`

---

### スコープ / 作業項目

#### 1. 配送結果画面のUI作成
- `app/delivery-result.tsx` をモーダル形式で作成
- Expo Routerのモーダルプレゼンテーション設定

#### 2. 配送結果の表示
- 走行距離（XX.X km）
- 所要時間（XX分）
- 平均速度（XX km/h）
- 獲得経験値（+XXX exp）

#### 3. レベルアップ表示
- レベルアップした場合は「Lv.5 → Lv.7」のように変化を表示
- 複数レベルアップの場合も対応
- レベルアップしていない場合は「Lv.5 (経験値 +100)」

#### 4. 進化バッジの表示
- 進化条件を満たした場合（レベル20または50）は「進化できます！」バッジを表示
- 「進化を見る」ボタンを配置（タップで進化アニメーション画面へ遷移）

#### 5. アニメーション
- 画面表示時に数値がカウントアップするアニメーション
- レベルアップ時にキラキラエフェクト

#### 6. ナビゲーション
- 「ホームへ戻る」ボタンでホーム画面へ遷移
- 「進化を見る」ボタン（進化条件を満たした場合のみ表示）で進化画面へ遷移

---

- [ ] 走行距離・所要時間・平均速度・獲得経験値を表示できる
- [ ] レベルアップした場合は「Lv.X → Lv.Y」のように変化を表示できる

### テスト観点

#### 手動テスト
- 配送終了後、配送結果画面がモーダル形式で表示されることを確認
- 走行距離・所要時間・平均速度・獲得経験値が正しく表示されることを確認
- レベルアップした場合、変化が表示されることを確認
- レベル19→20の配送で「進化できます！」バッジが表示されることを確認
- 「ホームへ戻る」ボタンでホーム画面へ遷移することを確認
- 「進化を見る」ボタンで進化画面へ遷移することを確認（Phase 2で実装）

#### 検証方法
1. 配送を開始
2. 一定距離移動（テスト用に短距離でOK）
3. 配送終了ボタンをタップ
4. 配送結果画面が表示されることを確認
5. 表示内容が正しいことを確認
6. 「ホームへ戻る」ボタンをタップし、ホーム画面へ遷移することを確認
