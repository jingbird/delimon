# Issue #22: 履歴画面の実装（後半）

### 背景 / 目的
期間別の統計カードと走行距離グラフを実装し、ユーザーが自分の配送実績を視覚的に確認できるようにする。

- **依存**: #21
- **ラベル**: `frontend`, `feature`

---

### スコープ / 作業項目

#### 1. react-native-chart-kitの導入
- `react-native-chart-kit@6.12+` をインストール
- react-native-svgも一緒にインストール（依存関係）

#### 2. 期間別統計カードの作成
- 上部に統計カード（3つの指標）を表示:
  - **合計走行距離**（例: 125.5 km）
  - **配送回数**（例: 15回）
  - **獲得経験値**（例: 12,550 exp）
- 期間タブ切り替えで統計も更新

#### 3. 走行距離グラフの作成
- 棒グラフ（BarChart）で走行距離を表示
- 横軸: 日付（例: 11/1, 11/2, ...）
- 縦軸: 距離（km）
- 期間タブ切り替えでグラフも更新

#### 4. グラフのスタイリング
- プロジェクトのカラーテーマに合わせた配色
- グラフのラベル、グリッド線、凡例を設定

#### 5. パフォーマンス最適化
- グラフがスムーズにレンダリング（500ms以内）
- データが多い場合はサンプリング（例: 今月の場合は週ごとに集計）

---

### ゴール / 完了条件（Acceptance Criteria）

- [ ] `react-native-chart-kit@6.12+` がインストール済み
- [ ] 期間別統計カード（合計走行距離・配送回数・獲得経験値）を上部に表示できる
- [ ] 走行距離グラフを棒グラフで表示できる（横軸: 日付、縦軸: 距離）
- [ ] 期間タブ切り替えでグラフと統計が更新される
- [ ] グラフがスムーズにレンダリング（500ms以内）

---

### テスト観点

#### 手動テスト
- 統計カードに合計走行距離・配送回数・獲得経験値が表示されることを確認
- 走行距離グラフが棒グラフで表示されることを確認
- 期間タブを切り替え、統計カードとグラフが更新されることを確認
- グラフのレンダリングがスムーズであることを確認（500ms以内）
- 配送履歴がない場合、グラフが空になることを確認

#### 検証方法
1. 履歴画面で統計カードが表示されることを確認
2. 走行距離グラフが表示されることを確認
3. 期間タブを「今週」→「今月」→「全期間」と切り替え
4. 統計カードとグラフが正しく更新されることを確認
5. グラフのレンダリング時間を確認（目視で500ms以内）
