# Issue #16: チュートリアル画面の実装

### 背景 / 目的
初回起動時の4画面スライド形式のチュートリアルを実装し、アプリの使い方をユーザーに説明する。

- **依存**: #7
- **ラベル**: `frontend`, `feature`

---

### スコープ / 作業項目

#### 1. チュートリアル画面のUI作成
- `app/tutorial.tsx` を作成
- react-native-pager-viewまたはFlatListでスライド実装

#### 2. 4画面のコンテンツ作成
- **画面1**: アプリの概要（「配送をゲーム化」）
- **画面2**: キャラクター育成の説明（「モンスターを育てよう」）
- **画面3**: GPS計測の説明（「走った距離で経験値ゲット」）
- **画面4**: アカウント登録への誘導（「さあ、始めよう！」）

#### 3. インジケーターの表示
- 現在位置を示すインジケーター（●●○○）を下部に表示
- スワイプで画面切り替え時にインジケーターも更新

#### 4. ナビゲーション
- 4画面目に「アカウント登録」ボタンを配置
- タップでサインアップ画面へ遷移
- スキップボタンは配置しない（必ず4画面見る仕様）

#### 5. 画像・イラスト
- 各画面にキャラクター画像・説明イラストを配置
- assetsフォルダに画像を追加

#### 6. ローカルストレージ管理
- チュートリアル完了フラグをローカルストレージに保存
- 2回目以降の起動時はチュートリアルをスキップ

---

### ゴール / 完了条件（Acceptance Criteria）

- [ ] `app/tutorial.tsx` で4画面のスライド実装完了（スワイプで切り替え）
- [ ] 各画面にキャラクター画像・説明文を表示できる
- [ ] インジケーター（●●○○）で現在位置を表示できる
- [ ] 4画面目に「アカウント登録」ボタンを配置、タップでサインアップ画面へ遷移できる
- [ ] スキップ不可（必ず4画面見る仕様）

---

### テスト観点

#### 手動テスト
- 初回起動時にチュートリアル画面が表示されることを確認
- スワイプで画面が切り替わることを確認
- インジケーターが正しく更新されることを確認
- 4画面目の「アカウント登録」ボタンでサインアップ画面へ遷移することを確認
- 2回目以降の起動時にチュートリアルがスキップされることを確認

#### 検証方法
1. アプリを初回起動
2. チュートリアル画面が表示されることを確認
3. スワイプで4画面すべてを確認
4. インジケーターが正しく更新されることを確認
5. 「アカウント登録」ボタンをタップし、サインアップ画面へ遷移することを確認
6. アプリを再起動し、チュートリアルがスキップされることを確認
